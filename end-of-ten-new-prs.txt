The stack (merge top-down from #22; GitHub retargets each next PR as you merge and delete branches):

- #22 rename-branding — user-facing "Easy!Appointments" becomes "OpenAppointments" everywhere (locales, views, mailers; GPL headers kept), links point at your repo, upstream premium/blog/social sections removed, default colour #39824f via seeds + a data migration from the white sentinel.
- #23 single-name-field — one name column replaces first/last (reversible backfill migration, ~50-file sweep incl. all the wizard/backend JS). API keeps EA v1 compat: firstName carries the full name, lastName is empty, input joins both, sort/search still work.
- #24 wizard-service-first — the wizard's first step splits into a service page then a provider page (5 frames); ?first=provider plus a "Select Provider First" link swap the order, with cross-filtering both ways. Also delivers the project's first Capybara system tests, which caught two latent bugs (missing available_languages JS var crashed the language popover on every public page; missing first_weekday fixture).
- #25 cards-pictures — Active Storage pictures on providers/secretaries/admins/services/categories with upload controls on the backend forms, and a booking_display_mode setting: cards mode shows category cards revealing service cards, provider cards, pictures on top. Storage persists at /app/data/storage on Cloudron.
- #26 phone-or-email — new default: a booking needs a phone number OR an email, each individually optional; enforced server-side (the old port only validated client-side) and in the wizard, toggle on the booking settings page.
- #27 turnstile — captcha_provider setting: ALTCHA (default) or Cloudflare Turnstile with site/secret keys, widget on the final step, server-side siteverify.
- #28 import-page — the Python pipeline ported into the app (TenToEight::Extract/Load): upload the raw 10to8 export, dry-run analyze, pick record types and appointment window, background import with progress polling, idempotent matching. Plus the reset section: typing RESET wipes business data while keeping admins and settings. Fixtures are synthetic — no PII in the repo.
- #29 outline-theme — new theme using coloured borders instead of solid fills (outline buttons, transparent badges/cards, white header with primary border), auto-listed in the theme picker.
- #30 iframe-embed — embedding toggle + allowed origin: booking flow gets a frame-ancestors CSP while the backend stays SAMEORIGIN, cookies go SameSite=None only on embedded paths, and the new Embedding settings page generates the copy-paste iframe + auto-resize snippet.
- #31 app-icon — openout-booking-icon-outline.svg everywhere: SVG favicons with regenerated favicon.ico, 192px logo (pages, backend, mail CID stays PNG), PWA icons, social card, and a 256px icon.png wired into CloudronManifest for the Cloudron dashboard. The README's new Icons section documents every site.
